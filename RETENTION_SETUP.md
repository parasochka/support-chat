# Retention-бот «Ника» — инструкция по настройке

Пошаговая настройка после того, как код задеплоен. Три части: **Telegram**,
**Railway**, **админка/бот**. В конце — примеры сообщений для публикации.

> Всё, кроме серверных секретов из Railway, настраивается **на продукт** в админке
> (раздел **Retention · Telegram**). Один сервис обслуживает много брендов —
> у каждого продукта свой бот, канал, медиатека, менеджеры и retention-KB.

---

## 1. Telegram — что создать

### 1.1. Бот у @BotFather
1. Открой [@BotFather](https://t.me/BotFather) → `/newbot`.
2. Задай имя (например `Nika Casino`) и username (например `nika_casino_bot`).
3. Скопируй **токен** вида `123456:ABC-DEF...` — понадобится в админке.
4. Полезные команды BotFather (необязательно, но желательно):
   - `/setdescription` — текст, который виден до старта чата (см. примеры ниже).
   - `/setabouttext` — короткое «о боте».
   - `/setuserpic` — аватар Ники.
   - `/setprivacy` → **Disable** — чтобы бот видел все сообщения в личке (для лички
     это и так работает, но лучше выключить приватность явно).
   - Команды меню можно **не** задавать: вход только по диплинку с сайта.

### 1.2. Канал (гейт подписки)
1. Создай Telegram-**канал** (публичный, с `@username`, или приватный с invite-ссылкой).
2. Добавь **бота администратором канала** (Manage channel → Administrators → Add).
   Это обязательно: без прав админа `getChatMember` не работает и гейт подписки не пройдёт.
3. Узнай **channel_id**:
   - публичный канал: `@my_channel`;
   - приватный: числовой id вида `-1001234567890` (можно получить, переслав пост канала
     боту вроде `@getidsbot`, или из логов после первого апдейта).
4. Запомни **URL канала** (`https://t.me/my_channel` или invite-ссылку) — кнопка
   «Открыть канал» в гейте ведёт на него.

## 2. Railway — переменные окружения и Volume

Задай в переменных сервиса (Settings → Variables). Секреты бота/каналов **не** сюда —
они на продукт в админке; здесь только деплой-уровень.

| Переменная | Обязательна | Значение |
|---|---|---|
| `TELEGRAM_WEBHOOK_SECRET` | желательно | Случайная строка (`openssl rand -hex 32`). Проверяется в заголовке вебхука, в URL её нет. Если не задать — упадёт на `SESSION_JWT_SECRET` (в проде задай отдельную). |
| `PUBLIC_BASE_URL` | да | Публичный адрес сервиса, например `https://chat.твойдомен.com`. Нужен, чтобы кнопка «Register webhook» в админке собрала правильный URL. |
| `RETENTION_MEDIA_DIR` | да (для медиа) | Путь монтирования **Volume**, например `/data/media`. Иначе фото не переживут редеплой. |
| `SECRETS_MASTER_KEY` | да (прод) | Ключ шифрования секретов продукта (токен бота, player-API-ключ). Отдельная сильная строка; смена ключа делает старые секреты нечитаемыми. |

**Volume (обязательно для медиатеки):**
1. Railway → сервис → **Volumes** → **New Volume**.
2. Mount path: тот же путь, что в `RETENTION_MEDIA_DIR` (например `/data/media`).
3. Redeploy. Загруженные фото теперь лежат на волюме; в Telegram после первой отправки
   кэшируется `file_id`, дальше пересылается без повторной загрузки.

> `python-multipart` уже добавлен в `requirements.txt` (нужен для загрузки фото).

## 3. Админка — настройка продукта

Зайди в админку (`/admin`), выбери нужный **продукт** в шапке (партнёр → продукт),
открой раздел **Retention · Telegram**.

### 3.1. Вкладка «Telegram config»
1. Отметь **Retention bot enabled**.
2. **Bot username** — `nika_casino_bot` (без `@`).
3. **Channel id** — `@my_channel` или `-1001234567890`.
4. **Channel URL** — `https://t.me/my_channel`.
5. **Player API URL** — если казино даёт endpoint профиля (см. §8 спеки); иначе пусто.
6. **Save config**.
7. Ниже, в «Secrets»: вставь **Telegram bot token** (и **Player API key**, если есть) →
   **Save secrets**.
8. Нажми **Register / refresh webhook with Telegram**. Должно ответить `Webhook set (bot @…)`.
   Проверить можно так: `https://api.telegram.org/bot<ТОКЕН>/getWebhookInfo`.

### 3.2. Вкладка «Retention KB»
Единая плоская база сценариев/офферов Ники (не 7 разделов поддержки). Добавь записи:
- **Title** — короткое имя сценария («Давно не заходил», «Спросил про бонус»).
- **Trigger / when** — когда применять (необязательно).
- **Body** — что Ника говорит/предлагает (конкретику — суммы/сроки/промо — бери из KB).
- **Links** — офиц. ссылки бренда (через запятую).

Примеры — в конце файла.

### 3.3. Вкладка «Media»
Загрузи фото (до ~500 на бренд). На каждое:
- **Description** — что на фото (обстановка, наряд, поза, настроение). Это **внутреннее**
  описание, игроку не показывается — по нему модель матчит запрос и пишет подпись.
- **Tags** — через запятую (`пляж, купальник, лето`).
- **level_min** — минимальный VIP-тир (число-ординал: индекс тира в списке `vip_tiers`,
  по умолчанию `none=0, bronze=1, silver=2, gold=3, platinum=4, diamond=5`).
- **stage** — ступень откровенности (1 — обычные, дальше по нарастающей).
- **Category** — необязательно.

### 3.4. Вкладка «Managers»
Пул живых менеджеров (для эскалированного входа). Добавь `Display name` + `@username`.
Раздача — round-robin, липкая (вернувшийся попадает к тому же). Отпуск — выключи флагом.

### 3.5. Вкладка «Settings» (группа `retention`)
JSON-редактор порогов (значения на продукт). Дефолты:
```json
{
  "daily_photo_cap": 10,
  "proactive_photo_cooldown_msgs": 6,
  "candidate_list_size": 6,
  "stage_advance_msgs": [20, 45, 80],
  "stage_advance_min_hours": 24,
  "max_stage": 4,
  "max_stage_by_tier": {"none": 2, "bronze": 2, "silver": 3, "gold": 4, "platinum": 4, "diamond": 4},
  "vip_tiers": ["none", "bronze", "silver", "gold", "platinum", "diamond"],
  "nonce_ttl_sec": 120,
  "profile_pull_ttl_sec": 3600
}
```
> `vip_tiers` — порядок тиров: индекс тира = ординал для `level_min`. Названия должны
> совпадать (в нижнем регистре) с тем, что приходит в `vip_level` профиля игрока.

### 3.6. Персона и тексты
- **Ретеншн-персона**: раздел **Prompt → Prompt variables** (persona_name, brand_name,
  tone_of_voice…) — общие для сайта и Telegram.
- **Тексты бота** (гейт подписки, кнопки меню, роутинг): раздел **Translations**, ключи
  с префиксом `rtn_*` (можно перевести на каждый язык). По умолчанию есть en/ru.

### 3.7. Вкладка «Analytics»
Вовлечённость, подписки, средняя ступень, отправленные фото, переходы к менеджерам,
таблица пользователей.

## 4. Интеграция сайта (две кнопки)

Сайт генерит диплинк per-user и открывает бота. Бэкенд-запрос:

```
POST /api/retention/deeplink
{
  "widget_key": "wk_...",              // тот же ключ продукта, что у виджета
  "signed_context": "<handshake>",     // подписанный профиль игрока (как в чате)
  "escalation": false                  // true — кнопка эскалации в чате поддержки
}
→ { "nonce": "r7Kx…", "deep_link": "https://t.me/nika_casino_bot?start=r7Kx…" }
```

- Кнопка **«Написать Нике»** → `escalation: false` → меню только с Никой.
- Кнопка **эскалации в чате поддержки** → `escalation: true` → в меню появляется
  «Перейти к менеджеру».
- Открой `deep_link` (в новой вкладке / через `tg://`). Nonce одноразовый, живёт
  `nonce_ttl_sec` секунд — генери его в момент клика.

> В dev (без `WIDGET_HANDSHAKE_SECRET` и без секрета продукта) вместо подписи берётся
> тест-профиль из админки — удобно проверить бота без интеграции сайта.

**Свежесть профиля** — три уровня, работают из коробки и мягко деградируют:
1. **Снапшот** — профиль с последнего диплинка (ничего настраивать не нужно).
2. **Ленивый pull** — если задан **Player API URL** (+ ключ в Secrets), сервис сам подтянет
   свежий профиль перед ходом, когда снапшот старше `profile_pull_ttl_sec` (по умолч. 1 ч).
   Казино реализует у себя `GET {player_api_url}?player_id=<id>` с `Authorization: Bearer <ключ>`,
   отвечает JSON с полями `_CONTEXT_FIELDS` (full_name, vip_level, balance, …).
3. **Push** — казино шлёт изменения нам: `POST /partner/{product_id}/player-update`,
   `Authorization: Bearer <handshake_secret продукта>`, тело `{player_id, ...изменившиеся поля}`.

Без Player API продукт живёт на снапшоте — схема деградирует, а не ломается.

## 5. Примеры сообщений для публикации

### 5.1. Описание бота (BotFather `/setdescription`)
```
Привет! Я Ника. Загляни — расскажу, что нового, покажу пару фото и подкину повод сыграть.
Чтобы начать, открой чат через свой кабинет на сайте.
```

### 5.2. About бота (BotFather `/setabouttext`)
```
Ника — твой персональный гид. Вход только с сайта.
```

### 5.3. Закреплённый пост в канале
> Пост в канале нужен как цель подписки; вход в бота — только с сайта, поэтому кнопку
> «Открыть бота» в канале ставить не обязательно (и органический вход всё равно закрыт).

```
Тут — самое интересное: свежие бонусы, турниры и поводы для игры. 
Подписывайся, чтобы ничего не пропустить, и продолжай общение с Никой в чате 💬
```

### 5.4. Примеры записей Retention KB
**Давно не заходил**
- Trigger: игрок не появлялся несколько дней
- Body: «Тепло поприветствуй, скажи, что скучала, спроси как настроение. Ненавязчиво
  напомни, что для вернувшихся часто есть приятные бонусы — конкретные условия только
  если они есть в KB.»
- Links: `https://brand.example/promo`

**Спросил про бонус**
- Trigger: игрок спрашивает про акции/бонусы
- Body: «Расскажи про актуальный бонус живо и по делу, конкретные суммы/условия — строго
  из KB. Позови попробовать.»
- Links: `https://brand.example/bonuses`

**Близок к депозиту**
- Trigger: игрок обсуждает пополнение/готов играть
- Body: «Поддержи азарт, поверь в его выигрыш, мягко подведи к игре. Без давления и без
  обещаний выигрыша.»
