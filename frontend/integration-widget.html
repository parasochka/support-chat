Виджет поддержки — подключение на сайт (CMS)

Как встроить готовый плавающий AI-чат поддержки на страницы казино. Только виджет: один сниппет + widget key. Обзор сервиса и архитектура — на обзорной странице; передача данных игрока (подписанный handshake) — на странице данных игрока; если вы строите собственный интерфейс вместо готового виджета — см. Chat API и свой UI. База URL во всех примерах — адрес этого сервиса.

1. Сниппет встраивания

Один сниппет на странице казино. Widget key берётся из админки: Structure → продукт → Widget key (там же есть кнопка «Copy embed snippet»).

<link rel="stylesheet" href="https://CHAT_HOST/widget.css">
<script type="module" src="https://CHAT_HOST/widget.js"
        data-widget-key="wk_XXXXXXXXXXXX"></script>
  • Widget key идентифицирует продукт (казино): по нему виджет получает свои темы, переводы, настройки и ключи. Ключ не секрет (лежит в HTML страницы), но неугадываем и его можно ротировать в админке.
  • Ключ также можно задать глобально: window.NPCHAT_WIDGET_KEY = "wk_…" до загрузки скрипта.
  • Без ключа виджет попадает в «default» продукт деплоя — режим одиночного продукта/дев-стенда.
  • Не забудьте добавить домен сайта в CORS_ALLOW_ORIGINS сервиса.
Ротация widget key немедленно отключает старые сниппеты — обновляйте встраивание на сайте синхронно с ротацией.

2. Программный монтаж с переопределениями

Альтернатива автомонтажу — импортировать mount и передать конфиг явно (например, чтобы подставить подписанный handshake с данными игрока):

import { mount } from "https://CHAT_HOST/widget.js";
mount({
  API_BASE: "https://CHAT_HOST",
  WIDGET_KEY: "wk_XXXXXXXXXXXX",
  SIGNED_CONTEXT: "<подписанный blob с данными игрока>", // см. /integration-data
});

Формат и подпись SIGNED_CONTEXT (HMAC-handshake бэкенда CMS, whitelist полей игрока, примеры кода на Node.js и Python) — на странице передачи данных игрока. Без handshake виджет работает анонимно (модель не знает имя/статус игрока).

3. Что виджет делает сам

Готовый виджет несёт всю клиентскую логику — на стороне сайта ничего дописывать не нужно:

  • Язык: интерфейс открывается на языке браузера мгновенно, а дальше и ответы, и хром следуют за языком, на котором реально пишет игрок.
  • Темы и переводы продукта подтягиваются без сессии (GET /api/chat/topics, GET /api/chat/i18n) и кэшируются — список тем рисуется без задержки; правки копирайта из админки доезжают без редеплоя виджета.
  • Сессия создаётся лениво — только когда игрок выбрал тему; «открыл-закрыл» посетитель не создаёт сессий и не тратит лимиты.
  • Автопереключение темы, пузыри-подсказки, завершение чата, эскалация (включая хендофф в Telegram-бота, когда у продукта включён ретеншн) — весь контракт Chat API обработан внутри.
  • Безопасный рендеринг: ответы ассистента — только белый список лёгкого Markdown, ввод игрока — строго литерально.
  • Мобильные: полноэкранный режим и подгонка под экранную клавиатуру через visualViewport.
Классы виджета имеют префикс npchat- и не конфликтуют со стилями хост-страницы; скрипт — vanilla ES-модуль без зависимостей и без билд-шага.

4. Cloudflare Turnstile — своя пара ключей на продукт/домен

Антибот-проверка — Cloudflare Turnstile в невидимом режиме: в дашборде Cloudflare создайте Turnstile-виджет с типом Invisible (игрок никогда не видит челлендж) и добавьте в него домен казино. Публичный site key задаётся на продукте в админке (Structure) — виджет подхватывает его сам через GET /api/chat/i18n, менять сниппет встраивания не нужно. Секрет Turnstile хранится шифрованно и записывается write-only через PUT /admin/products/{id}/secrets (наружу — только флаг «задан»). Деплой-переменные TURNSTILE_SITE_KEY/TURNSTILE_SECRET остаются лишь фолбэком для продуктов без своей пары.
Проверка — advisory (fail-open): если скрипт challenges.cloudflare.com у игрока заблокирован или не отвечает (такое бывает в отдельных регионах/сетях), виджет просто не отправляет токен, а сервис пропускает проверку и полагается на остальные слои антиспама (rate limit, cooldown, low-content, injection). Игрок никогда не теряет чат из-за недоступности Cloudflare; блокируется сессия только когда Cloudflare явно ответил «токен невалиден».

5. Чек-лист подключения

  • Создать продукт в админке (Structure) и скопировать embed-сниппет с его widget key.
  • Добавить домен сайта в CORS_ALLOW_ORIGINS сервиса.
  • Создать в Cloudflare Turnstile-виджет типа Invisible для домена казино и внести site key + секрет на продукт (или положиться на деплой-фолбэк).
  • Настроить подписанный handshake на бэкенде CMS, чтобы Ника знала имя/статус игрока — страница данных игрока.
  • Заполнить KB, промпт-переменные и переводы продукта в админке — чат готов.
